# v0.10 Operational Runbook

This runbook defines the manual validation and recovery checks that remain for
the v0.10 stabilization window. Record command output, timestamps, daemon logs,
and package versions for each section before marking a release candidate ready.

## Pre-Upgrade Baseline

Run these checks before replacing packages:

```bash
arca show compatibility
arca check upgrade
arca check upgrade backup /var/backups/arca-router/pre-upgrade.conf
arca show ha
arca show lcp
arca show class-of-service
arca show telemetry paths
systemctl status arca-routerd
```

Expected result:

- `arca check upgrade` reports no warnings, or every warning has a documented
  operator decision.
- The rollback archive has at least one valid entry.
- The configuration backup path is writable and the backup file does not
  already exist.
- HA, LCP, and QoS output has no unexpected errors for the deployed feature set.

## Management Security Smoke

For deployments using remote management, verify both the gRPC and Web/NMS API
security paths.

```bash
make security-audit
```

```bash
arca-routerd \
  --grpc-listen=127.0.0.1:9443 \
  --grpc-tls-cert=/etc/arca-router/tls/server.crt \
  --grpc-tls-key=/etc/arca-router/tls/server.key \
  --grpc-client-ca=/etc/arca-router/tls/client-ca.crt

arca \
  -grpc-address=127.0.0.1:9443 \
  -grpc-ca=/etc/arca-router/tls/ca.crt \
  -grpc-server-name=arca-routerd.example.net \
  -grpc-client-cert=/etc/arca-router/tls/client.crt \
  -grpc-client-key=/etc/arca-router/tls/client.key \
  show compatibility
```

```bash
install -m 0600 /dev/null /etc/arca-router/web-api-tokens
printf 'nms:read-only:%s\n' "${NMS_TOKEN}" > /etc/arca-router/web-api-tokens
systemctl restart arca-routerd
curl -fsS -H "Authorization: Bearer ${NMS_TOKEN}" \
  http://127.0.0.1:8080/api/nms/v1/status
```

Expected result:

- The installed service user, capabilities, config file modes, VPP socket, and
  FRR file-backend readiness match `docs/security-model.md`.
- gRPC connections fail without a trusted client certificate when mTLS is
  enabled.
- Web/NMS token files with permissions broader than `0600` are rejected at
  daemon startup.
- `read-only` tokens cannot call write endpoints, and `operator` or `admin`
  tokens can validate or commit configuration when intended.

## HA Failover Soak

Run this test on a clustered deployment with VRRP, etcd config sync, VPP LCP,
and FRR enabled.

1. Capture baseline output:

   ```bash
   arca show ha
   arca show vrrp
   arca show lcp
   curl -fsS http://127.0.0.1:9090/metrics > /tmp/arca-metrics-before.txt
   ```

2. Fail over the active node by stopping `arca-routerd` or isolating the
   management plane according to the lab plan.
3. Poll every 5 seconds for at least 10 minutes:

   ```bash
   arca show ha
   arca show vrrp
   arca show lcp
   ```

4. Restore the failed node and continue polling for another 10 minutes.

Expected result:

- Exactly one node is active for each VRRP group after convergence.
- `arca show ha` returns converged status after the planned disruption window.
- No stale config sync errors remain after the failed node rejoins.
- VPP LCP reconciliation reports no persistent inconsistencies.

## FRR and VPP Restart Recovery

Run restart recovery checks with traffic generators or protocol neighbors
available where possible.

```bash
systemctl restart frr
arca show bgp summary
arca show ospf neighbor
arca show vrrp
arca show bfd status

systemctl restart vpp
systemctl restart arca-routerd
arca show interfaces
arca show lcp
arca show routes
```

Expected result:

- FRR status commands recover without daemon restart after an FRR-only restart.
- After a VPP restart, restarting `arca-routerd` restores managed interface,
  LCP, route, and QoS intent state.
- BGP, OSPF, VRRP, and BFD sessions converge within the release candidate's
  documented target.

## Datastore Lock Recovery

Validate that daemon startup clears stale ephemeral state without corrupting
running configuration.

```bash
systemctl stop arca-routerd
cp /var/lib/arca-router/config.db /var/backups/arca-router/config.db.pre-lock-test
systemctl start arca-routerd
arca check upgrade
arca show configuration
```

For SQLite failure injection in a lab, create a candidate lock and candidate
text with a test build or SQLite client, stop the daemon without releasing the
lock, then restart `arca-routerd`.

Expected result:

- Startup removes stale candidate locks and candidate text.
- Running configuration and commit history remain intact.
- A new commit can be prepared and committed after cleanup.
- `arca check upgrade` still reports a valid rollback archive.

## Resource Leak and Churn

Run churn tests for at least 24 hours on release-candidate packages.

```bash
while true; do
  arca show interfaces >/dev/null 2>&1 || true
  arca show telemetry path /system count 1 >/dev/null 2>&1 || true
  arca show telemetry path /interfaces count 1 >/dev/null 2>&1 || true
  curl -fsS http://127.0.0.1:8080/api/status >/dev/null || true
  sleep 5
done
```

Collect:

```bash
systemctl status arca-routerd
journalctl -u arca-routerd --since '24 hours ago'
curl -fsS http://127.0.0.1:9090/metrics
ps -o pid,rss,vsz,etime,comm -C arca-routerd
```

Expected result:

- Memory growth stabilizes after warm-up.
- No file descriptor or socket growth persists across polling cycles.
- Web/NMS, CLI, NETCONF, and telemetry requests keep succeeding.

## Upgrade Rollback

If an upgraded package fails startup or post-upgrade validation:

1. Reinstall the previous package artifact or restore the previous package
   repository pin.
2. Restart `arca-routerd`.
3. Restore the pre-upgrade configuration only if running config was modified:

   ```bash
   arca show configuration rollback 1
   arca backup configuration rollback 1 /var/backups/arca-router/rollback-1.conf
   ```

4. Re-run `arca check upgrade` and record the failed package version, logs, and
   rollback evidence.

Expected result:

- The previous package starts with the preserved datastore.
- The operator can inspect and export rollback archive configuration.
- Any config restore is explicit and validated before commit.
