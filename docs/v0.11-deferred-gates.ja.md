# v0.11 Deferred Gates

この文書は、v0.10 から意図的に defer する gate の source of truth とする。v0.10 release は、release notes と sign-off record がこの文書を参照し、deferred work として明示している場合に限り、これらを未完了のまま sign-off できる。

## Lab Validation Gates

以下の gate は、VPP、FRR、HA peer、現実的な management traffic を含む lab 環境と release-candidate package が必要。

- HA failover soak。
- FRR / VPP restart recovery。
- 24-hour resource leak / churn testing。

v0.10 status:

- 手順は `docs/v0.10-operational-runbook.md` と `docs/v0.10-operational-runbook.ja.md` に文書化済み。
- lab が利用できない場合、実際の lab evidence は v0.10 sign-off の必須条件にしない。
- sign-off record には、これらを accepted v0.11 deferred gate として記録する。

v0.11 close criteria:

- package version、commit SHA、host OS、VPP version、FRR version、topology を記録する。
- planned disruption の前、最中、後の command output を添付する。
- test window 全体を覆う daemon log と Prometheus metrics を添付する。
- convergence time、transient warning、final owner sign-off を記録する。
- data-plane、control-plane、management-plane の説明できない failure は follow-up issue を作成する。

## NETCONF Startup Datastore

startup datastore support は v0.11 deferred gate ではなく、意図的な非対応とする。project policy として `urn:ietf:params:netconf:capability:startup:1.0` は advertise せず、`<startup/>` RPC target は `operation-not-supported` で拒否する。

将来の compatibility requirement がある場合のみ、この policy を明示的な design record で reopen する。その design では `startup` を latest running config の alias にしてはならない。独立 storage、migration、lock、validate、copy-config、delete-config、boot behavior、upgrade guard、external client evidence を定義した後でなければ `:startup` advertisement は検討しない。

## Standard NETCONF `:xpath`

v0.10 は、ncclient と libnetconf2-family evidence を sign-off に添付したうえで `urn:ietf:params:netconf:capability:xpath:1.0` を default で advertise する。

将来作業は default promotion ではなく maintenance とする。

- XPath behavior、client library、NETCONF framing を変更した場合は standard XPath evidence を再実行する。
- expression length、XML input size、selected element count、output size、depth、attribute count、evaluation timeout の DoS boundary を維持する。
- 各 release sign-off で client version、RPC payload、server `<hello>`、reply XML、interoperability deviation を記録する。

executable runbook として `docs/netconf-xpath-interop.md` を使う。

## v0.10 Sign-off Use

これらの gate なしで v0.10 を sign-off する場合:

- sign-off record に各 deferred gate を列挙する。
- v0.10 から defer して安全な理由を説明する。
- `arca show compatibility` と `docs/compatibility.md` が完了済み lab soak を support として主張していないことを確認する。startup datastore は意図的な非対応かつ非 advertise であること、standard `:xpath` evidence が link されていることを確認する。
- v0.10 tag 作成前に v0.11 tracking issue を作成または参照する。
