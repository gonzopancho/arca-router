# v0.10 リリース準備チェックリスト

このチェックリストは、v0.10 stabilization line の残りの release-readiness gate を記録する。互換性と support matrix の内容は、`docs/compatibility.md`、`internal/compat/policy.go`、`arca show compatibility` を source of truth とする。

## Documentation Freeze

release candidate を作成する前に、以下の document を freeze し、freeze 後の変更は release notes に記録する。

- `README.md`
- `CHANGELOG.md`
- `ROADMAP.md`
- `docs/compatibility.md`
- `docs/observability.md`
- `docs/security-model.md`
- `docs/v0.10-operational-runbook.md`
- `docs/v0.10-operational-runbook.ja.md`
- `docs/release-process.md`

freeze 条件:

- docs 内の CLI command が `arca -help` と interactive help に一致している。
- compatibility schema ID が `internal/compat/policy.go` と一致している。
- management security option が daemon / CLI flag と一致している。
- upgrade rollback guidance が `arca check upgrade` の出力と一致している。
- operational runbook command には release-candidate 実行 owner が記録されている。

## Support Matrix

v0.10 support matrix は `arca show compatibility` と `docs/compatibility.md` で公開する。

| Component | Supported | Required | Release Gate |
| --- | --- | --- | --- |
| VPP | 24.10+ | `vpp`, `vpp-plugin-core`, linux-cp plugin | installed package version と LCP/QoS capability output を operational runbook で確認する。 |
| FRR | 8.0+ | `bgpd`, `ospfd`, `ospf6d`, `zebra`, `staticd`, `mgmtd`, `vrrpd`, `bfdd` | restart recovery と routing protocol status command を operational runbook で確認する。 |
| SQLite datastore | schema 1-2 | current schema 2 | `arca check upgrade` を実行し、future schema version が古い binary で拒否されることを確認する。 |
| NETCONF | base:1.0 and base:1.1 | candidate, validate, rollback-on-error | startup datastore と standard `:xpath` が full compatibility gate を満たすまで advertise されないことを確認する。 |
| gRPC | `arca.router.v1` | default は Unix socket、optional で TCP/TLS/mTLS | local socket access と remote TLS/mTLS smoke check を確認する。 |
| NMS JSON | `arca.nms.*.v1` | additive schema | NMS collector example で status、telemetry catalog、schemas、snapshot endpoint を確認する。 |

## Release Candidate Evidence

v0.10 を release-ready と判断する前に、以下の evidence を添付または link する。

- `go test ./...`
- `go vet ./...`
- `make package-lint`
- DEB/RPM artifact の package build metadata。
- release-candidate package で実行した `arca show compatibility` の出力。
- release-candidate package で実行した `arca check upgrade` の出力。
- 完了済みの `docs/v0.10-operational-runbook.md` または `docs/v0.10-operational-runbook.ja.md` の evidence。
- 受け入れた warning と、その owner / rationale。

## Known Deferred Gates

以下は、明示的な仕様または lab execution が可能になるまで意図的に deferred とする。

- Formal NETCONF startup datastore support。
- Standard NETCONF `:xpath` capability advertisement。
- 各 release-candidate package に対する実際の HA failover soak、FRR/VPP restart recovery、24-hour churn result。
