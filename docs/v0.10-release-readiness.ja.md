# v0.10 リリース準備チェックリスト

このチェックリストは、v0.10 stabilization line の残りの release-readiness gate を記録する。互換性と support matrix の内容は、`docs/compatibility.md`、`internal/compat/policy.go`、`arca show compatibility` を source of truth とする。

## Documentation Freeze

release candidate を作成する前に、以下の document を freeze し、freeze 後の変更は release notes に記録する。

- `README.md`
- `README.ja.md`
- `CHANGELOG.md`
- `ROADMAP.md`
- `docs/compatibility.md`
- `docs/observability.md`
- `docs/security-model.md`
- `docs/v0.10-operational-runbook.md`
- `docs/v0.10-operational-runbook.ja.md`
- `docs/v0.10-release-signoff.md`
- `docs/v0.10-release-signoff.ja.md`
- `docs/v0.11-deferred-gates.md`
- `docs/v0.11-deferred-gates.ja.md`
- `docs/release-process.md`

freeze 条件:

- docs 内の CLI command が `arca -help` と interactive help に一致している。
- compatibility schema ID が `internal/compat/policy.go` と一致している。
- management security option が daemon / CLI flag と一致している。
- upgrade rollback guidance が `arca check upgrade` の出力と一致している。
- operational runbook command には release-candidate 実行 owner、または lab 実行ができない場合の v0.11 deferred owner が記録されている。

## Support Matrix

v0.10 support matrix は `arca show compatibility` と `docs/compatibility.md` で公開する。

| Component | Supported | Required | Release Gate |
| --- | --- | --- | --- |
| VPP | 24.10+ | `vpp`, `vpp-plugin-core`, linux-cp plugin | installed package version と LCP/QoS capability output を確認する。lab がない場合、restart/soak evidence は v0.11 に defer する。 |
| FRR | 8.0+ | `bgpd`, `ospfd`, `ospf6d`, `zebra`, `staticd`, `mgmtd`, `vrrpd`, `bfdd` | package と status command availability を確認する。lab がない場合、restart recovery evidence は v0.11 に defer する。 |
| SQLite datastore | schema 1-2 | current schema 2 | `arca check upgrade` を実行し、future schema version が古い binary で拒否されることを確認する。 |
| NETCONF | base:1.0 and base:1.1 | candidate, validate, rollback-on-error, standard `:xpath` | startup datastore が advertise されず拒否されることを確認する。standard `:xpath` は default で advertise され、ncclient/libnetconf2 evidence で裏付ける。 |
| gRPC | `arca.router.v1` | default は Unix socket、optional で TCP/TLS/mTLS | local socket access と remote TLS/mTLS smoke check を確認する。 |
| NMS JSON | `arca.nms.*.v1` | additive schema | NMS collector example で status、telemetry catalog、schemas、snapshot endpoint を確認する。 |

## Release Candidate Evidence

v0.10 を release-ready と判断する前に、以下の evidence を添付または link する。

- `go test ./...`
- `go vet ./...`
- `make package-lint`
- aggregate local readiness check を使う場合は `make release-check` の出力。
- local NETCONF evidence を sign-off に使う場合は
  `make release-evidence-check` の出力。
- installed release-candidate host が利用できる場合は `make security-audit`
  の出力。利用できない場合は、未取得の host evidence を accepted deferred
  evidence として記録する。
- DEB/RPM artifact の package build metadata。
- release-candidate package で実行した `arca show compatibility` の出力。
- release-candidate package で実行した `arca check upgrade` の出力。
- ncclient と libnetconf2 の default-mode `NETCONF Client Interoperability`
  artifact。standard `:xpath` が default で advertise され、XPath RPC が
  pass し、startup datastore request が拒否されることを示す。
- sign-off 用の dedicated standard XPath evidence として、
  `netconf-client-ncclient-standard-xpath-evidence` と
  `netconf-client-libnetconf2-standard-xpath-evidence`、または verify 済みの
  local `make netconf-standard-xpath-evidence` /
  `make netconf-standard-xpath-evidence-verify` output。
- `docs/v0.10-operational-runbook.md` または `docs/v0.10-operational-runbook.ja.md` のうち、実行可能な section の evidence。
- 受け入れた warning または deferred gate と、その owner / rationale。

## v0.11 Deferred Gates

以下は v0.11 に意図的に defer する。lab run が完了し evidence が添付されていない限り、v0.10 release evidence として扱わない。

- release-candidate package に対する HA failover soak、FRR/VPP restart recovery、24-hour churn result。

NETCONF startup datastore support は v0.10 で意図的に非対応とし、将来の
design で明示的に reopened されない限り v0.11 deferred gate ではない。

entry criteria、implementation semantics、gate を close するための evidence は `docs/v0.11-deferred-gates.md` を source of truth とする。

## Release Sign-off

final v0.10 tag を作成する前に、release PR、issue、または tag notes に release sign-off を記録する。template として `docs/v0.10-release-signoff.md` を使う。sign-off record には以下を含める。

- sign-off owner。
- date と timezone。
- commit SHA または release tag。
- test、package metadata、compatibility output、実行可能な runbook section の evidence link。
- 受け入れた warning と v0.11 deferred gate。
- 各 deferred gate の rationale。
